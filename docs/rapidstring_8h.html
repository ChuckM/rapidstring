◆ rs_free()

RS_API void rs_free ( rapidstring *  s)

Frees a string.

The string is in an invalid state after freeing. You must call rs_init(s) if you wish to reuse the same string after freeing.

A jump may be avoided by directly calling RS_FREE(s->heap.buffer); if the string is known to be on the heap. The additional one is for the null terminator, which is subtracted upon initial allocation.

Calling this fuction is unecessary if the string size is always smaller or equal to RS_STACK_CAPACITY.

Parameters
[in]sThe string to free.
Since
1.0.0
